# AGENTS.md

## Code originality and licensing

- Implement requirements using an original, clean-room design.
- Do not reproduce substantial code from repositories, tutorials,
  Stack Overflow, books, or other identifiable sources.
- Do not imitate distinctive naming, comments, structure, or implementation
  from an existing project.
- Prefer documented public APIs, official specifications, standard algorithms,
  and conventional design patterns.
- Do not add GPL, AGPL, SSPL, source-available, or similarly restrictive code
  or dependencies without explicit approval.
- Before adding a dependency, report its name, purpose, licence, maintenance
  status, and any significant licensing or security concern.
- Clearly identify code adapted from user-provided code or third-party sources.
- Preserve required copyright, attribution, and licence notices.
- Do not fabricate licences, authorship claims, copyright notices, or citations.
- When provenance or licensing is uncertain, stop and propose a clean-room
  alternative.
- For publication-ready changes, flag code requiring manual licence or
  similarity review.
